4. 多Agent协作方案,那种先做计划,然后逐步做这种
这种“先计划，后执行”的模式，在 Multi-Agent 领域通常被称为 **Plan-and-Execute** 架构。

通常会采用以下四个核心组件来实现：

* **Planner（计划者）：** 负责将用户的高级目标（Goal）分解成一系列具体的、可执行的步骤（Tasks）。
* **Executor（执行者）：** 负责根据计划中的当前任务，调用工具（Tools）完成具体操作。
* **Re-planner（重规划者）：** 关键环节。在每个步骤完成后，对比“预期结果”与“实际观察”，决定是继续执行下一步，还是修改剩余计划。
* **Task Tracker（任务追踪器）：** 维护全局状态（State），记录哪些任务已完成，哪些正在进行。

```python
# 定义全局状态
class PlanExecuteState(TypedDict):
    input: str            # 用户初始需求
    plan: List[str]       # 计划列表
    past_steps: List[str] # 已完成的步骤及结果
    response: str         # 最终输出

# 1. Planner 节点：生成任务清单
def plan_step(state: PlanExecuteState):
    # Prompt: "基于用户需求，分解成 1,2,3 步，输出为 JSON 列表"
    plan = planner_model.invoke(state["input"])
    return {"plan": plan}

# 2. Executor 节点：执行当前最顶层的任务
def execute_step(state: PlanExecuteState):
    task = state["plan"][0]
    # 调用工具或 Worker Agent 执行具体逻辑
    result = worker_agent.invoke(task)
    return {"past_steps": [f"Task: {task}, Result: {result}"]}

# 3. Re-planner 节点：判断是否需要调整
def replan_step(state: PlanExecuteState):
    # Prompt: "基于已完成的 {past_steps}，原计划 {plan} 是否需要修改？"
    # 如果任务全部完成，则输出最终回答；否则更新 plan
    update = replanner_model.invoke(state)
    if "FINAL_ANSWER" in update:
        return {"response": update["FINAL_ANSWER"]}
    return {"plan": update["new_plan"]}
```
