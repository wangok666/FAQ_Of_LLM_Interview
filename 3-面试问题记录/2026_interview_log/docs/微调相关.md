## 1. lora的原理?

假设原始权重矩阵为 $ W_0 \in \mathbb{R}^{d \times k} $（其中 $ d $ 为输出维度，$ k $ 为输入维度）。在全参数微调中，需要更新整个 $ \Delta W \in \mathbb{R}^{d \times k} $，参数量为 $ O(dk) $，这在大型模型中代价极高。

LoRA 的假设是：权重更新 $ \Delta W $ 具有**低内在秩**（intrinsic low rank），即可以用两个低秩矩阵的乘积近似表示：

$$
\Delta W = B A
$$

其中：
- $ A \in \mathbb{R}^{r \times k} $（下采样矩阵，rank $ r \ll \min(d, k) $）；
- $ B \in \mathbb{R}^{d \times r} $（上采样矩阵）。

更新后的权重为：

$$
W = W_0 + \Delta W = W_0 + B A
$$

前向传播过程变为：

$$
h = W_0 x + B (A x)
$$

（其中 $ x $ 为输入向量，$ h $ 为输出）。原始权重 $ W_0 $ 被冻结（不参与梯度更新），仅训练 $ A $ 和 $ B $。此时可训练参数量减少为 $ O(r(d + k)) $，通常比全参数微调减少 2–4 个数量级。

为稳定训练，常用缩放因子 $ \alpha $（scaling factor）：

$$
W = W_0 + \frac{\alpha}{r} B A
$$

#### 参数效率对比

- 全参数微调：训练参数 $ \approx dk $
- LoRA：训练参数 $ \approx r(d + k) $

例如，当 $ d = k = 4096 $、$ r = 8 $ 时，参数量可减少约 99.6%。

#### 最简全参数微调代码

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset

# ====================== 1. 定义简单模型 ======================
class SimpleModel(nn.Module):
    def __init__(self, input_dim=512, hidden_dim=1024, output_dim=10):
        super().__init__()
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        self.fc2 = nn.Linear(hidden_dim, output_dim)
        self.relu = nn.ReLU()

    def forward(self, x):
        x = self.relu(self.fc1(x))
        x = self.fc2(x)
        return x

# ====================== 2. 准备数据 ======================
batch_size = 32
num_samples = 1000

X = torch.randn(num_samples, 512)          # 输入特征
y = torch.randint(0, 10, (num_samples,))   # 分类标签

dataset = TensorDataset(X, y)
dataloader = DataLoader(dataset, batch_size=batch_size, shuffle=True)

# ====================== 3. 实例化模型、损失、优化器 ======================
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model = SimpleModel().to(device)

criterion = nn.CrossEntropyLoss()
optimizer = optim.AdamW(model.parameters(), lr=1e-4)   # 全参数都参与优化

# ====================== 4. 最简训练循环 ======================
num_epochs = 5

for epoch in range(num_epochs):
    model.train()
    total_loss = 0.0
    
    for batch_x, batch_y in dataloader:
        batch_x = batch_x.to(device)
        batch_y = batch_y.to(device)
        
        optimizer.zero_grad()
        
        outputs = model(batch_x)
        loss = criterion(outputs, batch_y)
        
        loss.backward()
        optimizer.step()
        
        total_loss += loss.item()
    
    avg_loss = total_loss / len(dataloader)
    print(f"Epoch [{epoch+1}/{num_epochs}], Loss: {avg_loss:.4f}")

print("全参数微调训练完成。")
```

## 3. 模型的增量训练? 怎么做

增量训练（Incremental Training），在大型语言模型（LLM）领域通常称为持续训练 或 连续学习

主要挑战是**灾难性遗忘**。常用缓解方法包括：

- **Replay（重放）**：在训练新数据时，混合少量旧数据（或用旧模型生成伪数据）。推荐比例 10%~50% 旧数据。
- **学习率调整（LR Re-warming & Re-decaying）**：新阶段开始时先升高学习率（warmup），然后逐步衰减，避免剧烈更新。
- **正则化 / 参数隔离**：冻结部分层、使用 LoRA 等参数高效方法、或对重要参数施加软掩码（soft-masking）。
- **模型合并（Model Merging）**：分别在不同数据上微调后，通过任务向量（Task Vector）或 SLERP 等方式合并。
- **混合训练**：全参数 + PEFT 结合，定期进行全参数 consolidate。

实际中最有效且简单的做法往往是 **Replay + LR 调度 + 较小学习率**。

#### 最简全参数增量训练代码示例

```python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments, Trainer
from datasets import load_dataset, concatenate_datasets

# 1. 加载已有模型和分词器（从上一次训练的 checkpoint）
model_name_or_path = "./results/checkpoint-1000"   # 替换为 checkpoint 路径
tokenizer = AutoTokenizer.from_pretrained(model_name_or_path)
model = AutoModelForCausalLM.from_pretrained(model_name_or_path)

# 2. 加载新数据（增量数据）
new_data = load_dataset("json", data_files="new_domain_data.jsonl", split="train")

# 混合少量旧数据（Replay）
old_data = load_dataset("json", data_files="old_data_sample.jsonl", split="train").select(range(5000))  # 采样一部分
combined_data = concatenate_datasets([new_data, old_data.shuffle().select(range(len(new_data)//5))])  # 例如 20% 旧数据

# 3. 数据预处理（causal LM 格式）
def tokenize_function(examples):
    return tokenizer(examples["text"], truncation=True, max_length=2048)

tokenized_dataset = combined_data.map(tokenize_function, batched=True, remove_columns=combined_data.column_names)

# 4. 训练参数（增量训练通常使用更小的学习率）
training_args = TrainingArguments(
    output_dir="./incremental_results",
    num_train_epochs=2,                    # 根据新数据量调整
    per_device_train_batch_size=4,         # 根据显存调整
    gradient_accumulation_steps=8,
    learning_rate=2e-5,                    # 比从头微调更小
    lr_scheduler_type="cosine",            # 或 warmup + cosine
    warmup_ratio=0.1,
    weight_decay=0.01,
    fp16=True,                             # 或 bf16
    logging_steps=10,
    save_steps=500,
    evaluation_strategy="no",              # 如有验证集可开启
    save_total_limit=3,
    report_to="none",
)

# 5. 启动 Trainer（全参数增量训练）
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized_dataset,
    tokenizer=tokenizer,
)

trainer.train(resume_from_checkpoint=False)   # False 表示从当前加载的权重继续训练新数据

print("增量训练完成。模型已保存至", training_args.output_dir)
```

**关键设置说明**：
- `learning_rate`：建议 1e-5 ~ 5e-5（比首次微调小 2~10 倍）。
- Replay：至少混合 10%~30% 的旧数据采样，可显著减轻遗忘。
- 如果显存不足，推荐改用 **LoRA 增量训练**（仅训练适配器）：


## 4.怎么多卡训练?

启动 Ray 集群

需要在两台机器上建立 Ray 的主从关系。确保两台机器可以互相通过 SSH 免密访问，并且安装了相同版本的依赖

**在机器 A (Head Node) 执行：**

```bash
ray start --head --port=6379
```

**在机器 B (Worker Node) 执行：**
（将 `<Head-Node-IP>` 替换为机器 A 的内网 IP）

```bash
ray start --address='<Head-Node-IP>:6379'
```

最优:`Ray + DeepSpeed + PEFT`

核心:
`from ray.train.huggingface.transformers import RayTrainReportCallback, prepare_trainer`
`trainer = prepare_trainer(trainer)`

```python
import ray
from ray.train.huggingface.transformers import RayTrainReportCallback, prepare_trainer
from ray.train.torch import TorchTrainer
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM, TrainingArguments, Trainer

def train_func():
    # 1. 加载模型
    model = AutoModelForCausalLM.from_pretrained("Qwen/Qwen2.5-72B", trust_remote_code=True)
    
    # 2. 配置 PEFT (LoRA)
    peft_config = LoraConfig(
        r=8, lora_alpha=32, target_modules=["q_proj", "v_proj"], lora_dropout=0.05, task_type="CAUSAL_LM"
    )
    model = get_peft_model(model, peft_config)

    # 3. 设置训练参数
    training_args = TrainingArguments(
        output_dir="./output",
        deepspeed="./ds_config.json", # 关键：调用 DeepSpeed
        per_device_train_batch_size=1,
        learning_rate=1e-4,
        bf16=True # A100 建议开启 bf16
    )

    trainer = Trainer(model=model, args=training_args, ...)
    
    # 4. 适配 Ray
    trainer = prepare_trainer(trainer)
    trainer.train()

# 5. 启动分布式执行器
trainer = TorchTrainer(
    train_func,
    scaling_config=ray.train.ScalingConfig(num_workers=8, use_gpu=True) # 8个worker对应8张卡
)
trainer.fit()
```

ZeRO-3 (Zero Redundancy Optimizer)：它会将模型参数、梯度和优化器状态分散到所有 8 张显存中。
