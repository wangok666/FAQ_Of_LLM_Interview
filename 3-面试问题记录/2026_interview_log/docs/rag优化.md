## 1. rag 有哪些优化手段?或者哪些阶段可以优化?

可从三大核心阶段展开：索引构建（Indexing）、检索召回（Retrieval）与答案生成（Generation）

```
┌─────────────────────────────────────┐
│  1. 索引构建阶段 (Indexing)          │
│     • 文档解析与预处理               │
│     • 文本分块策略 (Chunking)        │
│     • 向量化与索引构建               │
└─────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────┐
│  2. 检索召回阶段 (Retrieval)         │
│     • 查询理解与增强                 │
│     • 混合检索与重排序               │
│     • 元数据过滤与上下文增强         │
└─────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────┐
│  3. 生成阶段 (Generation)            │
│     • 提示词工程与上下文压缩         │
│     • 生成质量控制与幻觉抑制         │
└─────────────────────────────────────┘
```

#### 文档解析预处理 

- 标签

记录原始元数据（文件名、创建日期、来源、页码、作者等），这些元数据将在后续过滤和检索中发挥作用。

- 文本清洗与增强

规范化文本（去除特殊字符、统一标点、处理 Unicode）
去除无关元素（导航菜单）

#### 文本分块策略

- **固定大小分块（Fixed-Size Chunking）**：按预设字符数、词数或 Token 数（如 500 Tokens）均匀拆分，常引入 10%–20% 重叠（Overlap）以维持上下文连续性。优点是实现简单、速度快；缺点是易割裂句子或段落，导致语义不完整。适用于结构混乱的日志、代码等数据。

- **基于句子的分块（Sentence-Based Chunking）**：利用 NLP 工具（如 NLTK、spaCy）识别句子边界，再聚合若干完整句子形成块。保证语法完整性，语义连贯性优于固定大小分块。

- **递归字符分块（Recursive Character Text Splitting）**：按层次分隔符（段落 → 换行 → 空格等）递归拆分，在不超过最大块长度的前提下尽量保留自然边界。LangChain 默认推荐此方法，适应性较强。

- **语义分块（Semantic Chunking）**：计算句子嵌入（Embedding）的余弦相似度，当相似度低于阈值时切分，确保块内主题高度相关。适用于主题跳跃明显的文档，但计算成本较高。

- **基于文档结构的分块**：利用 Markdown、HTML 或 PDF 原生结构（如标题、章节、段落）进行拆分，保留层次信息。

其中PPL分块:

建议先采用递归字符分块作为基线,计算按句分开的前后相似度,如果小于阈值,则认为是新块了

#### 向量化与索引构建 

近似最近邻（ANN）索引核心类型
- 倒排文件索引（IVF, Inverted File）
- 分层可导航小世界图（HNSW, Hierarchical Navigable Small World）
- 乘积量化（PQ, Product Quantization）
```
是否需要强过滤条件？
├─ 是 → 优先 IVF 系列（粗筛+细搜，过滤性能稳定）
└─ 否 → 
    ├─ 追求极致Recall → HNSW
    ├─ 内存/成本敏感 → IVF_PQ / IVF_SQ8
    └─ 平衡方案 → IVF_FLAT + 适度nprobe
```

#### 查询理解与增强

- 重写问题
- 预设答案检索
- 查询分解
   * 将复杂、多跳（Multi-Hop）查询分解为多个简单子查询，逐一检索后再合成答案

#### 混合检索与重排序

```
典型流程：
1. 并行执行：
   - Dense: 向量相似度搜索（top-100）
   - Sparse: BM25/SPARSE编码关键词匹配（top-100）
2. 结果融合：
   - Reciprocal Rank Fusion (RRF)
   - 线性加权：score = α·vector_score + (1-α)·keyword_score
3. （可选）重排序：Cross-Encoder对top-50精排
```

```
1. doc_B (RRF得分: 0.0328)   # 两路均排名靠前，得分累加
2. doc_A (RRF得分: 0.0164)   # 仅向量检索第1名
3. doc_C (RRF得分: 0.0161)   # 仅向量检索第3名
4. doc_D (RRF得分: 0.0161)   # 仅BM25第2名
5. doc_E (RRF得分: 0.0159)   # 仅BM25第3名
```

## 4. 检索有哪些?bm25?

BM25 作为稀疏检索（Sparse Retrieval）的代表算法


#### (1) 逆文档频率（IDF）
$$
\text{IDF}(q_i) = \log \frac{N - n(q_i) + 0.5}{n(q_i) + 0.5}
$$
*   $N$：文档集合总数。
*   $n(q_i)$：包含词项 $q_i$ 的文档数量。
*   **原理**：衡量词项的区分能力。常见词（如“的”、“是”）区分度低，权重低；稀有词区分度高，权重高。

#### (2) 词频饱和度（Term Frequency Saturation）
公式中包含 $f(q_i, D)$ 的部分并非线性增长，而是呈现饱和趋势。
*   **原理**：随着词频增加，其对相关性的贡献逐渐减小。例如，一个词出现 1 次与出现 2 次的区别，远大于出现 20 次与 21 次的区别。
*   **参数 $k_1$**：控制饱和速度。$k_1$ 越大，词频的影响越接近线性；$k_1$ 越小，饱和度越高。通常取值在 1.2 至 2.0 之间。

#### (3) 文档长度归一化（Document Length Normalization）
公式中包含 $\frac{|D|}{\text{avgdl}}$ 的部分用于惩罚过长的文档。
*   **原理**：长文档包含更多词项，偶然匹配查询词的概率更高。若不进行惩罚，长文档将获得不公正的高分。
*   **参数 $b$**：控制长度归一化的强度。
    *   $b = 0$：不进行长度归一化。
    *   $b = 1$：完全归一化。
    *   通常取值在 0.5 至 0.75 之间。
