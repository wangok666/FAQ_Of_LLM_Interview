## 5. ppo, dpo, grpo,与 gspo 有什么区别?

ppo 需先训练独立的奖励模型（Reward Model，RM），基于人类偏好数据为响应打分；然后使用PPO进行策略优化，同时引入价值函数（Critic/Value Model）估计优势函数（Advantage），并通过KL散度惩罚确保策略更新稳定（避免偏离参考模型过远）。属于on-policy方法，需要在线生成样本。

dpo 无需显式奖励模型和RL循环，直接将偏好优化转化为分类问题。利用偏好对（chosen/rejected响应）推导隐式奖励，通过logistic损失直接优化策略模型，使其更倾向于生成“优选”响应。同时隐含KL约束。

*利用成对的偏好数据（preference pair），直接让模型“更喜欢”人类偏好的回答，同时“更不喜欢”被拒绝的回答**，整个过程像一个简单的二分类问题。

每一条训练样本包含以下三部分：
- **x**：输入提示（prompt）
- **y_w**（chosen / winner）：人类标注的**更好**、**更偏好**的回答
- **y_l**（rejected / loser）：人类标注的**更差**、**被拒绝**的回答

DPO 的目标就是：让模型在给定 x 时，生成 y_w 的概率相对提高，生成 y_l 的概率相对降低。

DPO 的训练目标是最小化以下负对数似然损失函数：

$$
\mathcal{L}_{\text{DPO}}(\pi_\theta; \pi_{\text{ref}}) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w | x)}{\pi_{\text{ref}}(y_w | x)} - \beta \log \frac{\pi_\theta(y_l | x)}{\pi_{\text{ref}}(y_l | x)} \right) \right]
$$

其中：
*   $\sigma(\cdot)$ 表示 Sigmoid 函数，即 $\sigma(z) = \frac{1}{1 + e^{-z}}$。
*   $\mathbb{E}$ 表示期望，采样自偏好数据集 $\mathcal{D}$。

| 符号 | 含义 |
| :--- | :--- |
| $\pi_\theta$ | 当前正在优化的策略模型（Policy Model），参数为 $\theta$。 |
| $\pi_{\text{ref}}$ | 参考模型（Reference Model），通常为冻结的预训练模型或指令微调模型。 |
| $x$ | 输入提示词（Prompt）。 |
| $y_w$ | 优选回答（Chosen/Winning response），人类偏好中较好的回答。 |
| $y_l$ | 劣选回答（Rejected/Losing response），人类偏好中较差的回答。 |
| $\beta$ | 温度系数（Temperature parameter），控制策略偏离参考模型的程度（相当于 KL 约束的强度）。 |
| $\mathcal{D}$ | 偏好数据集，包含三元组 $(x, y_w, y_l)$。 |

DPO 损失可以理解为以下逻辑：

> “对于每一个偏好对，我希望模型对 y_w 的相对概率（与参考模型相比）比对 y_l 的相对概率**更高**。如果当前模型还没有做到这一点，就给它一个惩罚，让它朝着这个方向调整。”



grpo 对于同一提示（prompt），采样一组（Group）多个响应，使用组内奖励的均值和标准差进行相对归一化计算优势（相对优势），从而替代价值函数（Critic）。可结合显式奖励模型（类似PPO），也可使用可验证奖励（Verifiable Reward，如数学/代码正确性）或直接偏好，兼具PPO的RL特性和DPO的简化性。


$$L^{CLIP}(\theta) = \mathbb{E}_t [ \min( r_t(\theta) \hat{A}_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_t ) ]$$

```python
def reward_function(generated_output_y, expected_output_y_prime):
    """
    极简奖励函数：
    - 完全包含期望的路由标签，给 1.0 分
    - 格式正确但标签错误，给 0.1 分（鼓励输出正确格式）
    - 瞎胡说，给 0.0 分
    """
    if expected_output_y_prime in generated_output_y:
        return 1.0
    elif "<tool>" in generated_output_y:
        return 0.1
    else:
        return 0.0
```

```python
import torch
import torch.nn.functional as F

# 假设模型和 tokenizer 已经加载: policy_model (Qwen2), ref_model (初始冻结的Qwen2)
# prompt = "帮我写一段快排代码"
# expected_y = "<tool>code</tool>"

G = 4 # Group 数量，每次对同一个问题采样 4 次
epsilon = 0.2 # PPO/GRPO 截断参数
beta = 0.01 # KL惩罚系数

for epoch in range(num_epochs):
    # 1. 组采样 (Group Sampling)
    # 针对当前 Prompt，让策略模型生成 G 个不同的回答 (开启温度采样)
    generated_outputs = [policy_model.generate(prompt, do_sample=True, temperature=0.8) for _ in range(G)]
    
    # 2. 计算奖励 (Reward)
    rewards = [reward_function(y, expected_y) for y in generated_outputs]
    rewards_tensor = torch.tensor(rewards)
    
    # 3. 计算相对优势 (Group Relative Advantage)
    mean_reward = rewards_tensor.mean()
    std_reward = rewards_tensor.std() + 1e-8 # 加 1e-8 防止除零
    advantages = (rewards_tensor - mean_reward) / std_reward 
    # 现在 advantages 是正负交替的值，均值为0
    
    loss = 0
    for i in range(G):
        # 4. 计算当前模型和参考模型的 log 概率
        log_prob_policy = get_log_probs(policy_model, prompt, generated_outputs[i])
        with torch.no_grad():
            log_prob_ref = get_log_probs(ref_model, prompt, generated_outputs[i])
            
        # 5. 计算重要性采样比例 (Ratio)
        ratio = torch.exp(log_prob_policy - log_prob_ref.detach())
        
        # 6. GRPO 截断目标 (Surrogate Objective)
        surr1 = ratio * advantages[i]
        surr2 = torch.clamp(ratio, 1.0 - epsilon, 1.0 + epsilon) * advantages[i]
        policy_loss = -torch.min(surr1, surr2) # 取最小后加负号，因为我们要最小化 loss
        
        # 7. 计算 KL 惩罚 (防止模型被训练得面目全非，忘记原来的语言能力)
        kl_div = log_prob_ref - log_prob_policy
        
        # 累加总 Loss
        loss += policy_loss + beta * kl_div
        
    # 8. 均值化 loss 并反向传播
    loss = loss / G
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()
    
    print(f"Epoch {epoch}, Loss: {loss.item()}")
```

```python
import torch
import torch.nn.functional as F

def dpo_loss(
    policy_model, ref_model,          # 策略模型与参考模型
    prompt_ids, chosen_ids, rejected_ids,  # token ids
    beta=0.1
):
    # 计算 log π_θ(y|x) 和 log π_ref(y|x)
    chosen_logp = get_log_probs(policy_model, prompt_ids, chosen_ids)      # sum over tokens
    rejected_logp = get_log_probs(policy_model, prompt_ids, rejected_ids)
    
    with torch.no_grad():
        ref_chosen_logp = get_log_probs(ref_model, prompt_ids, chosen_ids)
        ref_rejected_logp = get_log_probs(ref_model, prompt_ids, rejected_ids)
    
    # DPO loss
    logits = beta * (chosen_logp - ref_chosen_logp - (rejected_logp - ref_rejected_logp))
    loss = -F.logsigmoid(logits).mean()
    return loss
```
